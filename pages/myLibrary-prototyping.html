myLibrary

Projeto de prototipagem de um sistema web desktop para a gestão do acervo da biblioteca da Fatec de Araras. Foi desenvolvido utilizando o paradigma do Design Centrado no Usuário, onde por meio de ferramentas como brainstorm, pesquisa quantitativa por meio de um formulário e wireframes, são decididas características que o futuro sistemas terá. O protótipo foi desenvolvido em um software online e grátis chamado Figma.

Introdução

A Fatec foi inaugurada na cidade de Araras no ano de agosto de 2017 com o curso tecnólogo de Sistema para Internet. Para dar suporte aos alunos desse curso foi montada uma biblioteca com diversos títulos, ela realiza os empréstimos dos livros por meio de anotações em cartões com as informações referentes ao empréstimo. Conforme o corpo discente da instituição cresce, a demanda por empréstimos de livros da sua biblioteca também cresce e encontrar um meio de facilitar e agilizar o modo que esse processo é realizado é de extrema importância. Dentre as muitas alternativas para melhorar a eficiência das atividades da biblioteca da Fatec Araras, nos pareceu uma solução interessante realizar a prototipagem de um aplicativo web, a princípio no formato desktop com layout responsivo, que possibilitaria os alunos a pesquisarem os títulos dos livros existentes na biblioteca, realizarem reservas e a confirmação dos empréstimos e devoluções dos livros.

Revisão bibliográfica e problematização

Esse projeto teve início durante as aulas Projeto de Prototipagem e Testes de Usabilidade ministradas pelo professor Thiago Gonçalves Mendes onde ele pediu aos alunos para desenvolver em grupo a prototipagem de um sistema a fim fixar e exercitar os conteúdos adquiridos durantes as aulas. Antes do início do desenvolvimento propriamente dito desse novo sistema, foi realizado uma pesquisa em sites de vendas de livros, a fim de observar características que seriam interessantes incorporar no sistema.

Métodos e ferramentas

O desenvolvimento desse projeto seguiu o paradigma do Design Centrado no Usuário, que por meio de várias ferramentas que busca identificar as necessidades dos usuários e produzir um sistema facilmente utilizável e acessível, visando a satisfação do usuário.

Foi realizado um brainstorm entre os integrantes do grupo para gerar ideias de um sistema que seria útil para aplicar os conteúdos mostrados durante as aulas e de relevância real como produto.

Brainstorm é uma técnica para resolver problemas específicos, para desenvolver novas ideias e levantar informações. Ela é realizada por um grupo de pessoas onde cada uma delas expõe suas ideias sobre o tema em questão.

BRAINSTORM

Após o levantamento das ideias e de um estudo realizado sobre elas foi decidido desenvolver um sistema para a biblioteca da Fatec Araras e a partir disso foi possível iniciar a criação de wireframes para entender o layout do sistema e de um formulário por meio do google forms para realizar uma pesquisa quantitativa e coletar informações com possíveis usuários do sistema para aprimorar as funcionalidades contidas no projeto.

Relato do Processo

Para a prototipagem desse sistema foram realizadas as seguintes etapas:

  • Realização de um brainstorm entre os integrantes do grupo a fim de levantar informações relevantes para o desenvolvimento do projeto;
  • Desenvolvimento do wireframe para validar as informações coletadas durante o brainstorm e escolher o melhor formato para o site a ser criado;
  • Criação de um formulário online com questões relevantes ao projeto com o objetivo levantar informações com possíveis usuários do sistema;
  • Desenvolvimento do protótipo utilizando uma ferramenta online chamada Figma.

O processo para a realização da prototipagem de um sistema web iniciou com um brainstorm entre os integrantes do grupo para as seguintes tomadas de decisões:

Por meio desse brainstorm foi decidido que queria realizado a prototipagem de um sistema para a biblioteca da Fatec Araras que foi batizado de myLibrary.

Foi percebido a necessidade de duas personas que utilizaram o sistema, os aluna e o administrador. O sistema para uma biblioteca é formado por dois ambientes, aquele que o usuário final (no caso os alunos) utilizam e o backoffice que apenas os administradores têm acesso.

Resultados

O desenvolvimento da prototipagem do sistema myLibrary atendeu de forma mínima todos dos requisitos levantados durante o processo de brainstorm e de pesquisa para esse projeto. A prototipagem em si foi realizada de forma simples, mas completa e com certeza agregou muito conhecimento na forma de pensar o como o sistema vai funcionar e será utilizado pelo usuário.

Considerações

A princípio a ideia dessa prototipagem é referente apenas a um sistema web no formato desktop, mas atualmente a grande maioria das pessoas utilizam smartphones para acessar os sites e sistemas, então como uma possível nova feature seria importante a prototipagem desse mesmo sistema na versão mobile.

Para o real desenvolvimento do sistema a abordagem escolhida seria uma API em .net core, que seria consumida por uma aplicação web em Angular e como também seria interessante a versão mobile a API poderia ser consumida por uma aplicação Android, iOS ou Flutter.
