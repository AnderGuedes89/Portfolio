Introdução

A Fatec foi inaugurada na cidade de Araras no ano de agosto de 2017 com o curso tecnólogo de Sistema para Internet. Para dar suporte aos alunos desse curso foi montada uma biblioteca com diversos títulos, ela realiza os empréstimos dos livros por meio de anotações em cartões com as informações referentes ao empréstimo. Conforme o corpo discente de instituição de ensino cresce, a demanda por empréstimos de livros da sua biblioteca também cresce e encontrar um meio de facilitar e agilizar o modo que esse processo é realizado é de extrema importância. Dentre as muitas alternativas para melhorar a eficiência das atividades da biblioteca da Fatec Araras, nos pareceu uma solução interessante realizar a prototipagem de um aplicativo web, a princípio no formato desktop com layout responsivo, que possibilitaria os alunos a pesquisarem os títulos dos livros existentes na biblioteca, realizarem reservas e a confirmação dos empréstimos e devoluções dos livros.

Revisão bibliográfica e problematização

O desenvolvimento desse projeto teve início durante as aulas Projeto de Prototipagem e Testes de Usabilidade ministradas pelo professor Thiago Gonçalves Mendes onde ele pediu aos alunos para desenvolver em grupo a prototipagem de um sistema a fim fixar e exercitar os conteúdos adquiridos durantes as aulas. Antes do início do desenvolvimento propriamente dito desse novo sistema, foi realizado uma pesquisa em sites de vendas de livros, a fim de observar características que seriam interessantes incorporar no sistema.

Métodos e ferramentas

O desenvolvimento desse projeto seguiu o paradigma do Design Centrado no Usuário, que visa identificar as necessidades dos usuários e produzir um sistema facilmente utilizável e acessível, visando a satisfação do usuário.

Foi realizado um brainstorm entre os integrantes do grupo para gerar ideias de um sistema que seria útil para aplicar os conteúdos mostrados durante as aulas e de relevância real como produto.

Brainstorm é uma técnica para resolver problemas específicos, para desenvolver novas ideias e levantar informações. Ela é realizada por um grupo de pessoas onde cada uma delas expõe suas ideias sobre o tema em questão.

Após o levantamento das ideias e de um estudo realizado sobre elas foi decidido desenvolver um sistema para a biblioteca da Fatec Araras e a partir disso foi possível iniciar a criação de wireframes e de um formulário por meio do google forms para coletar informações com possíveis usuários do sistema para aprimorar as funcionalidades contidas no projeto. Esse formulário tem o objetivo de realizar uma pesquisa quantitativa.

Pesquisa quantitativa apresenta resultados que podem ser quantificados por dados numéricos que são relevantes para o estudo sendo realizado.
Wireframe é um protótipo ou versão primitiva de um projeto, ele é desenhado por meio de formas geométricas e linhas de maneira muito simples para representar o como o projeto foi estudado, é feito por meio de desenhos em papel e posteriormente reproduzido por meio de um software como Adobe XD ou Figma para validar a experiência do usuário.

Por meio dos wireframes para esse projeto foi definido que o desenvolvimento de seu protótipo seria realizado por meio do software online Figma que é utilizado para validar a experiência do usuário ao utilizar o sistema. Nesse projeto não foi realizado a validação das ações por meio de testes feitos com usuários, apenas pelos integrantes do grupo.

Relato do Processo

Foram realizadas as seguintes etapas para o desenvolvimento desse projeto para a aula Projeto de Prototipagem e Testes de Usabilidade.

  • Realização de um brainstorm entre os integrantes do grupo a fim de levantar informações relevantes para o desenvolvimento do projeto;
  • Criação de um formulário online com questões relevantes ao projeto com o objetivo levantar informações com possíveis usuários do sistema;
  • Quais os tipos de conteúdo ele gostaria de apresentar no site;
  • Desenvolvimento do wireframe para validar as informações coletadas durante o brainstorm e escolher o melhor formato para o site a ser criado;
  • Desenvolvimento do protótipo utilizando uma ferramenta online chamada Figma.
Icone sobre Briefing

O processo para a realização da prototipagem de um sistema web iniciou com um brainstorm entre os integrantes do grupo para as seguintes tomadas de decisões:

  • Qual o tema do sistema a ser realizada a prototipagem;
  • Como serão realizadas as ações antes da implantação do sistema;
  • Quem são os usuários desse sistema;
  • Quais são as ações que os usuários podem realizar;
  • Quais são as telas a serem desenvolvidas;
  • Quais dados cada tipo de usuário tem acesso;

Paralelamente foi criado um formulário utilizando o google forms com questões relevantes ao sistema a ser desenvolvido para coletar informações com possíveis usuários do sistema.

Para o melhor desenvolvimento do sistema para a biblioteca da Fatec Araras foram realizadas várias versões de wireframes para definir e validar as ações que os usuários poderiam realizar ao utilizar o sistema, além de auxiliar na melhor escolha para layout desse projeto.

A prototipagem do sistema foi realizada por meio de um software online chamado Figma onde são desenvolvidos os layouts das telas que estarão presente no sistema e a simulação das ações realizadas pelos usuários. Para esse projeto foi decidido que inicialmente seria realizada a prototipagem do sistema versão no formato desktop.

Resultados

Apesar desse projeto não estar hospedado em nenhum servidor, o desenvolvimento da One Page para o restaurante “Amor Amor” atendeu de forma mínima todos dos requisitos levantados para esse projeto, além de estar responsivo, pois atualmente a grande maioria das pessoas utilizam smartphones para acessar os sites e fazer suas pesquisas, então mesmo que a responsividade não ser um dos requisitos iniciais sua implementação foi de grande importância.

Considerações

A princípio a ideia dessa prototipagem é referente apenas a um sistema web no formato desktop, mas atualmente a grande maioria das pessoas utilizam smartphones para acessar os sites e sistemas, então como uma possível nova feature seria importante a prototipagem desse mesmo sistema na versão mobile.
