Restaurante Amor Amor

Atualmente estar presente na internet é de extrema importância para qualquer empresa, independente do seu porte suas atividades e seu tempo de existência, pois com isso é possível divulgar sua marca, seus produtos e principalmente conquistar novos clientes. Um site bem desenvolvido tem o poder de ser uma forma de cartão de visita de quaisquer estabelecimentos, onde os clientes recorrem para encontrar as informações mais importantes da empresa. Esse projeto tem como principal objetivo desenvolver um site para uma empresa localizada na cidade de Araras-SP e descrever como foram realizadas as etapas fundamentais para o melhor desenvolvimento do projeto.

Introdução

Quando uma empresa tem presença na internet por meio de um site, uma página ou mesmo um blog ela é capaz de propagar e divulgar sua marca e seus produtos para um número maior de pessoas e, por consequência, de possíveis novos clientes. Por esse motivo, esse projeto tem como objetivo divulgar o restaurante "Amor Amor", localizado na cidade de Araras, no interior de São Paulo, por meio de um site institucional.

Revisão bibliográfica e problematização

O desenvolvimento desse projeto iniciou durante as aulas Padrões de Projeto de Sítios Internet II ministradas pelo Dr. Paulo César de Macedo onde ele pediu aos alunos para desenvolver um site para uma empresa real que ainda não possuía presença online a fim de além de codificar as páginas ter a oportunidade de realizar um briefing com clientes

Métodos e ferramentas

O desenvolvimento desse projeto foi realizado por meio das seguintes etapas: Realização de um briefing com os responsáveis da empresa para coletar informações relevantes ao projeto e conhecer melhor o restaurante; Desenvolvimento do wireframe para validar as informações coletadas durante o briefing e escolher o melhor formato para o site a ser criado; Codificação de uma One Page utilizando HTML 5, CSS 3 e o Bootstrap que é o framework web de componentes de interface e front-end. Para o desenvolvimento desse projeto foi realizado um briefing com o responsável pelo restaurante “Amor Amor” a fim de coletar informações relevantes sobre a empresa, o mercado em que ela atua, seu público alvo e por consequência conhecer melhor o estabelecimento. Além de ser possível perceber suas preferências, objetivos e as ideias do cliente para o seu site. Após a coleta dessas informações e de um estudo realizado sobre elas foi possível iniciar a criação do wireframe para validar as ideias das telas e paletas de cores que estariam no site além das ações que os usuários poderiam realizar nele. Wireframe é um protótipo ou uma versão primitiva de um projeto, ele é desenhado por meio de formas geométricas e linhas de maneira muito simples. Esse protótipo é feito por meio de desenhos em papel e posteriormente reproduzido por meio de um software como Adobe XD ou Figma para validar as ações e a experiência do usuário. Por meio dos wireframes para esse projeto foi definido que seu desenvolvimento seria realizado por meio da codificação de um site em formato de uma One Page com design simples e objetivo com a utilização de HTML 5, CSS 3 e com componentes do Bootstrap como auxílio no design do layout da interface.

Relato do Processo

O processo para a realização do site do restaurante "Amor Amor" começou com a realização de um pequeno briefing com o proprietário do estabelecimento para coletar as seguintes informações: Qual o público-alvo que ele espera atingir com o site; Quais produtos e/ou serviços são oferecidos pelo restaurante; Quais os tipos de conteúdo ele gostaria de apresentar no site; Quais as ferramentas de marketing já são utilizadas na empresa e poderiam ser incorporadas ao novo site; Qual a imagem e as referências a serem transmitidas para os usuários; Quais as vantagens sobre seus concorrentes Após coletar essas informações com o proprietário do restaurante ficou claro que o desenvolvimento de um site seria importante para o fortalecimento da imagem da empresa e da marca “Amor Amor” em Araras e nas cidades da região. Na época ele possuía apenas uma conta no facebook pouca utilizada para fazer a divulgação do restaurante. Para o melhor desenvolvimento do site foram realizadas várias versões de wireframes para encontrar qual melhor se enquadraria no design, cores e nas ideias já existentes no restaurante. Como esse site precisa passar credibilidade ao cliente, disponibilizar as informações importantes da empresa como endereço, horário de funcionamento e como entrar em contato, foi escolhido um design simples e objetivo com as cores da marca (vermelho e preto) e imagens reais do restaurante reunidos no formato de One Page. One Page é um layout feito para exibir todo o conteúdo do site em apenas uma página, onde tem suas seções bem definidas para a exibição dos conteúdos. Quais as vantagens desse formato? Por que ele foi escolhido? A codificação da One Page foi realizada através de HTML 5 e CSS 3 e com a utilização do framework web para desenvolvimento front-end conhecido como Bootstrap que tem a finalidade de facilitar a codificação. O Bootstrap permitiu a inclusão de um carrossel de imagens do restaurante para mostrar aos futuros novos clientes o que esperar do ambiente, também foi utilizado cards e modais para mostrar os tipos de alimentos são vendidos ali além de uma área com efeito paralaxe no início da página. Por meio do briefing o proprietário do restaurante pediu para a paginar ter um mapa com a localização do restaurante. Com o auxílio do Bootstrap foi possível desenvolver a One Page de forma responsiva, para poder ser visualizada tanto em formato desktop quanto no formato mobile.

Resultados

Apesar desse projeto não estar hospedado em nenhum servidor, o desenvolvimento da One Page para o restaurante “Amor Amor” atendeu de forma mínima todos dos requisitos levantados para esse projeto, além de estar responsivo, pois atualmente a grande maioria das pessoas utilizam smartphones para acessar os sites e fazer suas pesquisas, então mesmo que a responsividade não ser um dos requisitos iniciais sua implementação foi de grande importância.

Considerações

Após o desenvolvimento dessa One Page foi possível perceber que apesar de conseguir finalizar todo o processo de criação, codificação e finalização da página existe muita a ser aprendido e treinado para os próximos projetos serem melhores.
